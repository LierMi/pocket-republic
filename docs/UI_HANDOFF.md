# UI Handoff: Pocket Republic 云上个人国度

交接日期：2026-07-15
产品主线：用户立宪，Agent 国民协作，Kite 国库在有限授权内支付。

## 先看这三份

1. [UI_ASSET_BRIEF.md](../assets/UI_ASSET_BRIEF.md)：七张图片的尺寸、构图、提示词。
2. [WORLD_BIBLE.md](./WORLD_BIBLE.md)：国家部门、角色和商业化。
3. [KITE_INTEGRATION.md](./KITE_INTEGRATION.md)：视觉词汇对应的真实技术对象。

## 产品叙事

Pocket Republic 不是多 Agent 聊天室，也不是传统钱包 Dashboard。

用户经历五章：

1. 建国：命名一片土地，写下当前使命与预算。
2. 立宪：编辑 Agent 必须遵守的规则。
3. 议案：Agent 为项目提出 API、数据或服务采购。
4. 国土：看见其他未来部门与商业入口。
5. 公报：追溯宪法判断、Kite Session 和 Receipt。

## 视觉方向

“云上童话中的严肃制度”。

- 70% 云上乐土和文学感。
- 20% 超现实童趣与奇异材质。
- 10% 数字舞台动效。

不要复制任何现成动画角色、服装、脸型、场景或标志性道具。灵感只保留“认真建立一个属于自己的国家”这份童年愿望。

## 页面重点

### 开屏

- 左侧标题和一句文学叙事。
- 只有一个主动作“让我的国度升空”。
- 右侧 ART-00 呈现尚未完成的云上土地。

### 建国

- ART-01 是国家种子，不是普通 Banner。
- 国家模板是性格选择。
- 用户输入使命、月度预算、单笔和高风险限额。
- ART-02 是七位国民群像。

### 宪法

- 宪法纸是真正可编辑表单。
- ART-03 提供仪式场景，不能抢输入区可读性。

### 财政议案

- 先展示“国库风门”：Passport、Session、x402、Receipt 四道门。
- ART-04 根据风险、Pending、Settled 有天气变化。
- 申请金额、批准金额、实际支付必须分开。
- 沙盒状态明确写“非链上”；真实结算才显示链上凭证。

### 国家地图

- ART-05 是最重要的长期世界观资产。
- 未来部门是产品与商业入口，不做 disabled 功能列表的感觉。

### 国家公报

- 像正式档案与诗意旅行凭证的结合。
- ART-06 是国玺与星轨装饰。
- JSON 是可展开的技术证据，不是页面主角。

## 图片替换

槽位固定为 `ART-00` 到 `ART-06`。替换后保留 `data-art-slot`、class、比例和 alt，删除 `.art-slot-note`。

必须检查：

- 1280 x 800
- 390 x 844
- 320 x 568
- reduced-motion

## 不要改动

- 五个产品入口和顺序。
- 宪法可编辑能力。
- 四道 Kite 国库风门。
- 沙盒与真实状态的视觉差异。
- 国家公报中的技术事实。

## 验收

打开页面五秒内，用户应该感到“我正在建立自己的国家”；进入财政议案十秒内，用户应该看懂“Agent 的权力如何被 Kite Session 限制并留下 Receipt”。
