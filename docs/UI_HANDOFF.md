交接日期：2026-07-15

1. [UI_ASSET_BRIEF.md](../assets/UI_ASSET_BRIEF.md)：七张图片的尺寸、构图、提示词。
2. [WORLD_BIBLE.md](./WORLD_BIBLE.md)：国家部门、角色和商业化。

## 产品叙事

Pocket Republic 不是多 Agent 聊天室，也不是传统钱包 Dashboard。

用户经历五章：

1. 建国：命名个人国度，选择国库宪法并写下当前使命与预算。
2. 立宪：编辑 Agent 必须遵守的规则。
3. 议案：Agent 为项目提出 API、数据或服务采购。
4. 国土：看见其他未来部门与商业入口。
5. 公报：追溯宪法判断、Kite Session 和 Receipt。

## 视觉方向

“云上童话中的严肃制度”。

- 70% 云上乐土和文学感。
- 20% 超现实童趣与奇异材质。
- 10% 数字舞台动效。

不要复制任何现成动画角色、服装、脸型、场景或标志性道具。灵感只保留“认真建立一个属于自己的国家”这份童年愿望。

## 页面重点

### 开屏

- 第一视觉必须同时出现 `Pocket Republic / 口袋共和国` 与“建立你的个人 AI 国度”。
- 主动作是“创建我的口袋国度”，次动作是“提交一笔财政议案”。
- ART-00 呈现完整个人国度，但左侧为产品定义、Kite 技术说明和双入口保留可读区域。

### 建国

- ART-01 是四套国度与国库宪法的概念版图，不是普通 Banner。
- 国家模板同时是性格选择与 Kite 钱包治理预设。
- 每张模板卡必须显示 USDC 指标、支付规则、审批路线与 Demo 场景。
- `Kite Treasury Policy Preview` 随模板切换，并明确区分 Sandbox 与 Kite Passport。
- 用户输入使命、月度预算、单笔和高风险限额。
- ART-02 是七位国民群像。

### 宪法

- 宪法纸是真正可编辑表单。
- ART-03 提供仪式场景，不能抢输入区可读性。

### 财政议案

- 先展示“国库风门”：Passport、Session、x402、Receipt 四道门。
- ART-04 根据风险、Pending、Settled 有天气变化。
- 申请金额、批准金额、预检报价和 Receipt 实付金额必须分开。
- 沙盒状态明确写“非链上”；真实结算才显示链上凭证。

### 国家地图

- ART-05 是最重要的长期世界观资产。
- 未来部门是产品与商业入口，不做 disabled 功能列表的感觉。
- 创作工坊标记“已接入国库试验”；心灵花园、Agent 大学、佣兵公会和道具铺统一标记 `Roadmap v0.2`。

### 国家公报

- 像正式档案与诗意旅行凭证的结合。
- ART-06 是国玺与星轨装饰。
- JSON 是可展开的技术证据，不是页面主角。
- 公报终端在沙盒模式明确写“非链上记录”；真实模式只展示 Kite 实际返回的 Agent、Session、Receipt 支付金额与结算引用。只有报价时不能写成 `Paid Amount`。

## 不要改动

- 五个产品入口和顺序。
- 宪法可编辑能力。
- 四道 Kite 国库风门。
- 沙盒与真实状态的视觉差异。
- 国家公报中的技术事实。

## 验收

打开页面五秒内，用户应该感到“我正在建立自己的国家”；进入财政议案十秒内，用户应该看懂“Agent 的权力如何被 Kite Session 限制并留下 Receipt”。
