---

## 1. 产品一句话

Pocket Republic 不是一个多 Agent 聊天室，而是一个个人 AI 国度。用户先建国、写宪法、激活 Agent 国民；当用户或 Agent 想花钱时，议案进入国民议会审议，最后由 Kite 国库执行通过治理的金额，并生成国家公报。

视觉不要做成纯技术证明台，也不要做成泛 fantasy 游戏页。应该像一个有童趣隐喻、但能真实管理 Agent 钱包的个人操作系统。

---

## 2. 当前页面结构

### 2.1 开屏页

目的：让用户先理解“个人国度”，而不是一进来就看 Kite 字段。

页面元素：

- 产品名：Pocket Republic
- 中文副标题：口袋共和国：由 AI Agent 治理的个人国度
- 主按钮：开始建国
- 次按钮：先看国库议案
- 四个承诺：建国 / 宪法 / 国民 / 国库

### 2.2 产品工作台

核心结构：

- 左侧：Agent 国民护照，展示 7 个国民的职位、权限、护照、信誉和忠诚对象。
- 顶部：我的国家，展示国家名、当前目标、国民数量、预算边界和当前国库。
- 主 tabs：建国 / 个人宪法 / 财政议案 / 国家地图 / 国家公报。
- 右侧 rail：Kite 国库连接、国家主线、触发条款、未来部门。

### 2.3 五个主视图

1. 建国  
   选择国家模板，输入目标和预算，点击“更新个人宪法”。

2. 个人宪法  
   展示 6 条可编辑条款。UI pass 可以强化“立宪者正在写规则”的仪式感。

3. 财政议案  
   用户提交支出议案，或选择预设议案。点击“运行国库审查”后，Agent 国民给出意见，Kite 国库执行批准金额。

4. 国家地图  
   展示当前可跑通的国库核心，以及创作工坊、心灵花园、学院、外交邮局、道具铺等未来入口。

5. 国家公报  
   展示最终 receipt、Kite-style trace JSON 和公报历史。它是产品里的“国库账本”，不是显性的评委检查器。

## 4. UI 优先级

1. 建国流程  
   必须让用户知道自己在创建一个国家，而不是填钱包参数。

2. 可编辑宪法  
   必须看起来像用户自己的规则，不能像静态说明卡。

3. Agent 国民护照  
   必须表达身份、职位、权限、信誉和“忠诚对象：宪法”。

4. 财政议案  
   必须让用户一眼看懂：申请金额、批准金额、为什么被限额、谁投了什么意见。

5. 国家地图  
   必须保留童趣想象力，同时明确当前 demo 只跑通 Kite 国库。

6. 国家公报  
   必须像执行凭证和历史账本，不要像为了展示技术而硬塞 JSON。

---

## 5. 不建议改动的产品边界

黑客松阶段不要把心灵部、创作部、学院做成完整功能。它们应该是可见入口和商业想象力，但第一条可运行主线必须是：

> 建国 -> 写宪法 -> 提交财政议案 -> 国民议会审议 -> Kite 国库执行 -> 国家公报

---

## 6. 首屏验收标准

打开页面 5 秒内应该能看懂：

- 产品叫 Pocket Republic / 口袋共和国。
- 这是一个个人 AI 国度，不是多 Agent 作业。
- 用户可以写宪法。
- Agent 是国民，有职位和护照。
- Kite 是国库，负责钱包授权和支付执行。
